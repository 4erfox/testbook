# JWT Аутентификация — Инструкция

## Установка зависимостей

```bash
npm install jsonwebtoken bcryptjs
```

## Настройка пароля

1. Сгенерируй хэш своего пароля:
```bash
node -e "const b=require('bcryptjs');console.log(b.hashSync('МОЙ_ПАРОЛЬ',10))"
```

2. Открой `admin.env` и заполни:
```
JWT_SECRET=длинная_случайная_строка_минимум_32_символа
ADMIN_USERNAME=admin
ADMIN_PASSWORD_HASH=хэш_из_шага_выше
JWT_EXPIRES_IN=8h
```

3. Добавь `admin.env` в `.gitignore`!

## Куда положить файлы

```
твой-проект/
├── admin-server.js     ← заменить (корень проекта)
├── admin.env           ← новый файл (корень проекта)
├── .gitignore          ← добавить строку: admin.env
└── public/
    └── admin/
        ├── bridge.js       ← заменить
        └── admin-panel.js  ← заменить
```

## Запуск

```bash
node admin-server.js
```

## Как работает

1. Браузер подключается по WebSocket
2. Панель показывает форму логина
3. Вводишь логин/пароль → сервер проверяет через bcrypt
4. Сервер выдаёт JWT токен (живёт 8 часов)
5. Токен сохраняется в localStorage
6. При следующем открытии — токен проверяется автоматически
7. Все команды (readFile, writeFile и т.д.) проверяют наличие валидного токена

## Безопасность

- Пароли хранятся только в виде bcrypt-хэша
- JWT подписывается секретным ключом из `admin.env`
- Защита от timing attacks при проверке логина
- Сервер слушает только `127.0.0.1` (localhost)
